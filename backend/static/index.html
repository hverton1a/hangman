Hangman API

Developed with Python, using the FastAPI framework and Docker.

There is a simple ReactJs available at Vercel, PLAY NOW!

Have a Look at Github for the source code and files.

For the Doc/Swagger Here.

The words are scraped from the web and stored in a dockerized Mysql Server DB.

Designed to be multi-language, storing words in a original and sanitarized form, only dependant to be alphabetical.

Although only word in portuguese are available at the moment.

The Container were deployed at a GCP - Google Cloud - VPS instance, with docker-compose.

To Handle Tsl and serve HTTPS/WSS for secure connetions was spun up a NGINX as a Docker container as a reverse proxy with a Certbot as a Docker container to handle with LetsEncript the certificates.

The workflow was dockerized, coded remotely inside the container.

Features:

  • Modular Word Scraper

    Get words from dictionary site or api on the web, implemented with recipes, being flexible and ease to aquire/update the words from another sources, just add a recipe and the scraper will deal with it.

  • Dockerized

    Speeding up the deployment and scale

  • Designed to be Alphabet dependant, but not language dependant

    The Word Table was designed to receive sanitarized and original word forms, and the language was designed one-to-many relationship.

It´s still in development, at the moment it´s not possivel to create or log in a user, also the game statistics was not implemented.
